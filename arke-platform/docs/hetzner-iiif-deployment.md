# Hetzner IIIF Deployment Layout

This is the recommended production layout for serving extracted site illustrations on a Hetzner server with IIIF.

## Stack

- `nginx` as reverse proxy and static file server
- `Cantaloupe 5` as the IIIF Image API server
- extracted master images stored on local disk
- static IIIF Presentation manifests generated by the image pipeline

This is intentionally lighter than a full repository platform. It fits the current ArkeOpen use case better: stable site illustration delivery plus Mirador-compatible manifests.

## Recommended paths

On the Hetzner host:

- master images: `/srv/iiif/images/drive-site-images`
- static manifests: `/var/www/iiif-manifests/sites`
- Cantaloupe config: `/etc/cantaloupe/cantaloupe.properties`
- Cantaloupe runtime user: `cantaloupe`

Public URLs:

- IIIF Image API base: `https://iiif.yourdomain.org/iiif/3`
- site manifests base: `https://iiif.yourdomain.org/manifests/sites`

## nginx layout

Recommended vhost:

- `https://iiif.yourdomain.org/iiif/3/...` -> reverse proxy to Cantaloupe
- `https://iiif.yourdomain.org/manifests/sites/...` -> static JSON files from `/var/www/iiif-manifests/sites`

Example shape:

```nginx
server {
    server_name iiif.yourdomain.org;

    location /iiif/3/ {
        proxy_pass http://127.0.0.1:8182/iiif/3/;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /manifests/sites/ {
        alias /var/www/iiif-manifests/sites/;
        add_header Access-Control-Allow-Origin *;
        types { application/json json; }
    }
}
```

## Cantaloupe shape

Use `FilesystemSource` with a path prefix pointing to the extracted image root.

Minimal shape:

```properties
endpoint.iiif.3.enabled = true
source.static = FilesystemSource
FilesystemSource.BasicLookupStrategy.path_prefix = /srv/iiif/images
processor.selection_strategy = ManualSelectionStrategy
processor.ManualSelectionStrategy.ica_retrieval_strategy = DownloadStrategy
```

The pipeline generates identifiers like:

- `drive-19425655a763/p013-01-liang-bua.jpg`

Those become IIIF Image API service URLs like:

- `https://iiif.yourdomain.org/iiif/3/drive-19425655a763%2Fp013-01-liang-bua.jpg`

## Root-relative mode

The local pipeline can now emit root-relative URLs:

- manifests: `/iiif/manifests/sites/...`
- IIIF image service: `/iiif/3/...`
- static image fallback: `/assets/drive-site-images/...`

This is the preferred mode while you are still local, because the CSV does not depend on `localhost` or a future domain name.

When you later move to a host server, you do not need to rebuild the CSV if the host exposes the same URL paths.

## Pipeline settings

Before running the pipeline for production, set:

```bash
export IIIF_MANIFEST_BASE_URL="/iiif/manifests/sites"
export IIIF_IMAGE_SERVICE_BASE_URL="/iiif/3"
export PUBLIC_IMAGE_URL_BASE="/assets/drive-site-images"
```

For a production deployment, keep the same path layout on the host:

- `/iiif/manifests/sites/...`
- `/iiif/3/...`
- `/assets/drive-site-images/...`

Then transfer the generated files:

- generated images from local output -> `/srv/iiif/images/drive-site-images`
- generated manifests -> `/var/www/iiif-manifests/sites`

Typical transfer procedure later:

1. Sync image masters to the IIIF image root on the host
2. Sync manifest JSON to the host manifest directory
3. Configure nginx so `/iiif/manifests/sites/` serves those JSON files
4. Configure nginx + Cantaloupe so `/iiif/3/` points to the IIIF Image API
5. Optionally expose `/assets/drive-site-images/` as a plain static fallback

## ArkeOpen import result

The builder writes `WEB_IMAGES` as one manifest URL per site, for example:

- `/iiif/manifests/sites/drive-19425655a763.json`

That is better than raw image URLs because:

- it is stable
- it is IIIF-native
- Mirador can load it directly
- multiple images per site stay grouped together

## Deployment flow

1. Run the pipeline locally with production base URLs.
2. Sync generated image files to `/srv/iiif/images/drive-site-images`.
3. Sync generated manifest JSON to `/var/www/iiif-manifests/sites`.
4. Reload nginx and Cantaloupe.
5. Import the CSV with manifest URLs in `WEB_IMAGES`.
